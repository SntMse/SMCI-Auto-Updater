# SMCI Auto Updater

## 📌 概要 (Overview)
Google スプレッドシート（SMCI管理表）の顧客/コンタクト情報を、Google コンタクト（連絡先）へ自動的に同期・更新する Google Apps Script (GAS) システムです。
GitHubを「正（マスター）」のコードベースとし、GAS上からボタン一つで最新コードをプルできるモダンなデプロイ環境を構築しています。

## 🏗 システム構成 (Architecture)
* **データベース:** Google Spreadsheet (複数ファイル対応)
* **実行環境:** Google Apps Script (V8) + Google Cloud Project連携
* **出力先:** Google Contacts API (People API v1)
* **コード管理:** GitHub (Private Repository 推奨)

## 📁 ファイル構成 (Files)
システムは以下の5つのファイルで構成されています。
1. `Config_Main.gs` : 環境変数、バージョン定義（`SCRIPT_VERSION`）、設定値
2. `Logic_Process.gs` : メインの同期ロジック、データ整形、差分チェック
3. `API_Manager.gs` : Google Contacts API (People API) との通信処理
4. `Utils_Helpers.gs` : 日付変換（皇紀・年なし対応など）や文字列処理の汎用関数
5. `_GitHub_Sync.gs` : GitHubから最新コードを取得し、GAS自身を上書きするデプロイツール

---

## 🚀 使い方・実行コマンド (Usage)
GASエディタ上部のプルダウンから関数を選択し、「実行」を押します。

### 1. コードの更新（GitHub同期）
* **実行する関数:** `pullFromGitHub_New` (`_GitHub_Sync.gs`内)
* **動作:** GitHubの `main` ブランチから最新のコード（5ファイル）を取得し、GASのコードを全上書きします。
* **用途:** GitHub上でコードを修正した後のデプロイに使用します。

### 2. コンタクト同期の実行（手動実行 / トリガー実行用）
* **実行する関数:** `main` または `syncContacts` 等（※実際のメイン関数名に合わせてください）
* **動作:** スプレッドシートのデータを読み込み、Googleコンタクトへ同期します。

---

## 🤖 自動更新の仕様 (Auto-Update Logic)
Google Contacts API の制限（タイムアウトやクオータ超過）を防ぐため、システムは「分割バッチ処理（レジューム機能）」を採用しています。

### 状態管理（レジューム機能）
GASの「スクリプトプロパティ」に以下の値を保存し、どこまで処理したかを記憶しています。
* `SYNC_FILE_INDEX`: 現在処理中のスプレッドシートIDの配列インデックス
* `SYNC_ROW_INDEX`: 現在処理中のスプレッドシートの行番号（例: 168行目）

### 処理フローと頻度
1. **処理のチャンク化:** 1回の実行につき、指定された行数（例: 50行〜100行）だけを処理します。
2. **自動実行（トリガー）:** GASの「トリガー」機能を使用し、**「〇分おき」** にメイン関数を定期実行します。
3. **継続処理:** 前回の `SYNC_ROW_INDEX` の続きから処理を再開し、最後まで到達したらインデックスをリセットして次のシート（または最初）に戻ります。

---

## 📝 スプレッドシートの入力ルール
特にエラーが起きやすい「日付（誕生日など）」の入力ルールをシステム側で吸収しています。

* **通常の日付:** `2000/10/15` 👉 そのまま反映
* **年なし（月日のみ）:** `'10/15` （先頭にアポストロフィ） 👉 年不明の「10月15日」として反映
* **皇紀:** `2660/10/15` 👉 西暦に自動変換され「2000年10月15日」として反映

## 🔐 環境構築・初期設定 (Setup)
このコードを別の環境で動かす場合、コード内に直接パスワード等は記述せず、GASの **「スクリプトプロパティ（金庫）」** に以下の設定値を追加する必要があります。

### 必要なスクリプトプロパティ
* `GITHUB_USER`: GitHubのユーザー名（例: SntMse）
* `GITHUB_REPO`: リポジトリ名（例: SMCI-Auto-Updater）
* `GITHUB_TOKEN`: GitHubのパーソナルアクセストークン（`repo`権限必須）
* `TARGET_SS_IDS`: 同期対象のスプレッドシートID（カンマ区切り、またはJSON形式）

### GCPプロジェクトの紐付け（重要）
`_GitHub_Sync.gs` を動作させるためには、デフォルトのGCPプロジェクトではなく、標準のGCPプロジェクトを作成し、**「Google Apps Script API」** を有効化した上で、GASのプロジェクト設定からプロジェクト番号を紐付ける必要があります（403エラー回避のため）。
